# sitio-sistemas-distribuidos
Sitio web sobre Sistemas Distribuidos - Actividad práctica
1. ¿Qué ventajas encuentras en trabajar directamente en la nube con GitHub?
Trabajar directamente en la nube con GitHub tiene varias ventajas importantes. Permite acceder al proyecto desde cualquier lugar sin depender de un equipo específico, facilita el control de versiones y el seguimiento de cambios, y permite colaborar con otras personas en tiempo real. Además, elimina la necesidad de instalar herramientas complejas en el computador y simplifica la publicación del sitio mediante servicios como GitHub Pages. También mejora la organización del proyecto, ya que todo queda centralizado en un repositorio.
2. ¿Qué desafíos tuviste al crear el sitio sin usar herramientas locales?
Uno de los principales desafíos al crear el sitio sin herramientas locales es la falta de retroalimentación inmediata, ya que no siempre se puede ver el resultado en tiempo real como en un entorno de desarrollo local. También puede ser más difícil depurar errores, especialmente de diseño o estructura, porque no se cuenta con herramientas avanzadas del navegador configuradas en un entorno de desarrollo completo. Otro reto es la dependencia de la conexión a internet, ya que sin ella no se puede trabajar ni guardar cambios.
3. ¿Cómo podrías mejorar este sitio web en el futuro?
Este sitio web podría mejorarse en varios aspectos. Se podrían agregar animaciones y transiciones para hacer la interfaz más atractiva, implementar un diseño más avanzado con frameworks como Bootstrap o Tailwind, y añadir funcionalidad dinámica utilizando JavaScript. También sería útil integrar un backend o servicios externos para que el formulario de contacto funcione realmente. Además, se podría optimizar la experiencia en dispositivos móviles, mejorar la accesibilidad y agregar más contenido interactivo o educativo sobre sistemas distribuidos.
4. ¿Qué aprendiste sobre Sistemas Distribuidos al crear este sitio?
Al crear este sitio se puede comprender mejor el concepto de sistemas distribuidos en la práctica, especialmente en el uso de herramientas como GitHub, que funcionan como plataformas distribuidas para el control de versiones. Se aprende cómo múltiples usuarios pueden trabajar sobre un mismo proyecto desde diferentes ubicaciones, cómo se gestionan los cambios y cómo se mantiene la consistencia del sistema. También se refuerza la idea de que los sistemas distribuidos buscan escalabilidad, disponibilidad y colaboración eficiente entre diferentes componentes.
